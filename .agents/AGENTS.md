# Project Custom Rules & Guidelines

- **Educational Focus (Critical):** Whenever introducing, installing, or writing code using a technology, package, database, design pattern, or architecture:
  1. Clearly explain **what** we are using.
  2. Explain **why** we are using it.
  3. List the **alternatives** for that tool/pattern.
  4. Explain **why we are NOT using those alternatives** in this specific context.
